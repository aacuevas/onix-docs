Software Guide¶

Currently, the ONIX project is targeting Bonsai for its software needs. Bonsai is extremely good at dealing with the asynchronous and heterogeneous data that ONIX hardware produces. Have a look at the Bonsai.ONIX Reference for detailed information on each of the elements of Bonsai.ONIX library. See Example Bonsai Workflows for example workflows that work out of the box with ONIX hardware. For those interested in using the Open Ephys GUI for acquisition, have a look at the Open Ephys GUI page.

Note

The API is meant to be used! If you want to incorporate ONIX hardware functionality into your acquisition software, please get in touch. It’s not as hard as you might think :).
